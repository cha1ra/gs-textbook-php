テンプレートリテラル

テンプレートリテラルとは
  • 文字列に変数を埋め込む便利な方法
  • ES6から追加された機能で、従来の文字列結合よりもスッキリとコードが書ける
従来の書き方
console.log('私の名前は' + name + 'です');
長くて読みにくい...
テンプレートリテラル
console.log(`私の名前は${name}です`);
スッキリと読みやすい！
基本的な使い方
const name = 'ラッコ';
const age = 5;

// 従来の文字列結合
console.log('私の名前は' + name + 'で、年齢は' + age + '歳です');

// テンプレートリテラル
console.log(`私の名前は${name}で、年齢は${age}歳です`);
バッククォート（`）の入力方法
  • 日本語配列キーボード: Shift + @
  • US配列キーボード: alt(option) + ~

配列

配列とは
  • 配列は複数の値を順番に格納できるデータ構造
  • 一つの変数で多くのデータを管理できる

従来の方法で曜日を管理する場合：

// 変数だけで曜日を作る場合（非効率）
let sunday = "日";
let monday = "月";
let tuesday = "火";
let wednesday = "水";
let thursday = "木";
let friday = "金";
let saturday = "土";

配列を使った効率的な方法：

// スッキリ！
let weekdays = ["日", "月", "火", "水", "木", "金", "土"];

各値には0から始まる番号（インデックス）が自動的に割り当てられる

配列の基本文法
// 配列の宣言
const fruits = ["りんご", "バナナ", "オレンジ"];

// 配列の中身を取り出す（インデックスは0から開始）
console.log(fruits[0]); // "りんご"
console.log(fruits[1]); // "バナナ"
console.log(fruits[2]); // "オレンジ"

// 配列の長さを取得
console.log(fruits.length); // 3

// 混在したデータ型も格納可能
const mixedArray = ["文字列", 123, true, null];
インデックスは0から始まる

プログラミングでは配列のインデックス（番号）は0から始まります。これはメモリアドレスのオフセットを表しているためです。

const colors = ["赤", "青", "緑"];
// インデックス:   0     1     2
console.log(colors[0]); // "赤"
console.log(colors[2]); // "緑"
console.log(colors[3]); // undefined（存在しない）
演習 おみくじアプリを配列で作ってみよう
// おみくじの結果を配列で管理
const omikuji = ["大吉", "中吉", "小吉", "末吉", "凶"];

// ランダムなインデックスを生成（0〜4）
const randomIndex = Math.floor(Math.random() * omikuji.length);

// 結果を表示
console.log(`今日の運勢は${omikuji[randomIndex]}です！`);
配列の長さを使った動的な処理

omikuji.lengthを使うことで、配列の要素数が変わっても自動的に対応できます。これはプログラミングの重要なテクニックですね！

配列操作メソッド

配列にはデータを追加・削除するための便利なメソッドが用意されています。

const fruits = ["りんご", "バナナ"];

// 末尾に要素を追加
fruits.push("オレンジ");
console.log(fruits); // ["りんご", "バナナ", "オレンジ"]

// 先頭に要素を追加
fruits.unshift("いちご");
console.log(fruits); // ["いちご", "りんご", "バナナ", "オレンジ"]

// 末尾の要素を削除
const lastFruit = fruits.pop();
console.log(lastFruit); // "オレンジ"
console.log(fruits); // ["いちご", "りんご", "バナナ"]

// 先頭の要素を削除
const firstFruit = fruits.shift();
console.log(firstFruit); // "いちご"
console.log(fruits); // ["りんご", "バナナ"]
push() / unshift()
要素を配列に追加する（末尾/先頭）
pop() / shift()
要素を配列から削除し、その値を返す（末尾/先頭）
その他の代表的な配列操作メソッド

要素の検索・確認

const numbers = [1, 2, 3, 4, 5];
const fruits = ["りんご", "バナナ", "オレンジ"];

// indexOf() - 要素のインデックスを取得（見つからない場合は-1）
console.log(fruits.indexOf("バナナ")); // 1
console.log(fruits.indexOf("ぶどう")); // -1

// includes() - 要素が含まれているかチェック
console.log(fruits.includes("りんご")); // true
console.log(fruits.includes("ぶどう")); // false

// find() - 条件に合う最初の要素を取得
const found = numbers.find(num => num > 3);
console.log(found); // 4

配列の変形・フィルタリング

const scores = [85, 92, 78, 96, 88];

// map() - 各要素を変換した新しい配列を作成
const doubled = scores.map(score => score * 2);
console.log(doubled); // [170, 184, 156, 192, 176]

// filter() - 条件に合う要素だけの新しい配列を作成
const highScores = scores.filter(score => score >= 90);
console.log(highScores); // [92, 96]

// sort() - 配列をソート（元の配列を変更）
const sortedScores = [...scores].sort((a, b) => b - a); // 降順
console.log(sortedScores); // [96, 92, 88, 85, 78]

配列の結合・分割

const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];

// concat() - 配列を結合した新しい配列を作成
const combined = arr1.concat(arr2);
console.log(combined); // [1, 2, 3, 4, 5, 6]

// slice() - 配列の一部を切り出した新しい配列を作成
const sliced = combined.slice(1, 4);
console.log(sliced); // [2, 3, 4]

// join() - 配列の要素を文字列として結合
const joined = arr1.join("-");
console.log(joined); // "1-2-3"
検索・確認系
• indexOf() - インデックス取得
• includes() - 存在確認
• find() - 条件検索
変形・加工系
• map() - 要素変換
• filter() - 条件フィルタ
• sort() - ソート
結合・分割系
• concat() - 配列結合
• slice() - 部分切り出し
• join() - 文字列結合
追加・削除系
• push() / pop() - 末尾操作
• unshift() / shift() - 先頭操作
配列メソッドは実際に使いながら覚えよう！

ここで紹介したのは配列操作の一部です。jQueryのメソッドと同じように、実際にコードを書きながら覚えていくのが一番効果的！ まずはpush()、indexOf()、includes()あたりから使ってみて、慣れてきたらmap()やfilter()にも挑戦してみましょう。

補足 なぜconstで宣言した配列にpush()できるのか？

constの定義は 「再代入を禁止する」 ことです。配列の中身を変更することは再代入ではないため、push()やpop()などの操作が可能です。

const arr = [1, 2, 3];

// ✅ 中身の変更は可能
arr.push(4);
console.log(arr); // [1, 2, 3, 4]

// ❌ 再代入は禁止
// arr = [4, 5, 6]; // TypeError: Assignment to constant variable.
constの本当の意味

constは「値を変更できない」ではなく「参照を変更できない」という意味！配列やオブジェクトの場合、参照先（メモリ上のアドレス）は変わらないため、中身の変更は許可されているのです

for文（繰り返し処理）

for文とは
  • 指定した回数だけ処理を繰り返すための制御構文
  • 同じ処理を何度も実行したい場合に使用

例: 5回繰り返し処理を行う場合

図表を読み込み中...
基本的な文法
// for文の基本構文
for (初期値; 条件; 増減処理) {
  // 繰り返し実行される処理
}

// 具体例：0から4まで表示
for (let i = 0; i < 5; i++) {
  console.log(i); // 0, 1, 2, 3, 4 が順番に表示される
}
初期値（let i = 0）
ループ開始時に一度だけ実行される
条件（i < 5）
ループを続けるかどうかを判定
増減処理（i++）
各ループの最後に実行される
i++の意味

i++はi = i + 1と同じ。
以下の3つの書き方は同じ意味です！

i = i + 1;
i += 1;
i++;

See the Pen for文の理解 by cha1ra (@cha1ra) on CodePen.

配列とfor文の組み合わせ
  • for文は配列の全要素を処理する際によく使用される
const fruits = ["りんご", "バナナ", "オレンジ", "ぶどう"];

// 配列の全要素を表示
for (let i = 0; i < fruits.length; i++) {
  console.log(`${i}番目: ${fruits[i]}`);
}
// 出力:
// 0番目: りんご
// 1番目: バナナ
// 2番目: オレンジ
// 3番目: ぶどう
配列の長さ（length）を活用

fruits.lengthを使うことで、配列の要素数が変わっても自動的に対応できますね！

モダンな配列処理：forEach
  • 実際の開発では、従来のfor文よりもforEachメソッドなどが利用される
  • より直感的で読みやすいコードになる
const fruits = ["りんご", "バナナ", "オレンジ"];

// 従来のfor文
for (let i = 0; i < fruits.length; i++) {
  console.log(fruits[i]);
}

// forEach（推奨）
fruits.forEach(function(fruit, index) {
  console.log(`${index}: ${fruit}`);
});

// アロー関数を使ったより短い書き方
fruits.forEach((fruit, index) => {
  console.log(`${index}: ${fruit}`);
});
演習 食べ物リストをfor文で表示してみよう！

HTML構造：

<ul id="food-options"></ul>

JavaScript実装（jQuery使用）

const foods = [
    'ラーメン',
    '寿司',
    'カレー',

    'ハンバーガー',

配列とfor文の基本パターン

配列の要素数だけ繰り返し処理を行うのは、プログラミングの基本パターン！
i < todos.lengthで配列の長さまで繰り返し、インデックスiは0から始まることがポイントです

パターン2: forEachメソッドを使用

foods.forEach((food) => {

localStorage

localStorageとは
  • Webブラウザにデータを永続的に保存できる仕組み
  • ページを閉じたり、ブラウザを再起動してもデータが残り続ける
データ保存
ユーザー設定
入力内容
永続保存
ページ更新後も
データが残る
ブラウザ保存
サーバー不要で
ローカル管理
localStorageが使われる場面
ユーザー設定

テーマ（ダーク/ライト）、言語設定、フォントサイズなどの個人設定を保存

ショッピングカート

ECサイトで商品をカートに追加した状態を保持。ページを離れても商品が残る

フォーム下書き

長いフォームの入力途中でページを離れても、入力内容が自動保存される

閲覧履歴

最近見た記事や商品の履歴を保存し、「最近見たアイテム」として表示

localStorageの特徴
永続化
ブラウザを閉じてもデータが残る
ドメイン単位
サイトごとに独立したストレージ
容量制限
ブラウザにより2M〜10M程度
キー・バリュー形式
文字列のペアでデータを保存
基本的な使い方
// データの保存
localStorage.setItem('userName', '田中太郎');
localStorage.setItem('userAge', '25');

// データの取得
const name = localStorage.getItem('userName');
const age = localStorage.getItem('userAge');
console.log(name); // "田中太郎"
console.log(age);  // "25"

// データの削除
localStorage.removeItem('userName');

// 全データの削除
localStorage.clear();

// データの個数を取得
console.log(localStorage.length);

// インデックスでキーを取得
const firstKey = localStorage.key(0);
文字列での保存

localStorageはすべてのデータを文字列として保存します。数値や真偽値も文字列として扱われるため、必要に応じて型変換が必要です！

localStorage.setItem('count', 123);
const count = localStorage.getItem('count');
console.log(typeof count); // "string"
console.log(count + 1);    // "1231" (文字列結合)

// 数値として使用する場合は変換が必要
const countNumber = Number(count);
console.log(countNumber + 1); // 124

補足 JSONを使った複雑なデータの保存
  • オブジェクトや配列をlocalStorageに保存する場合は、JSON形式に変換して保存する
// オブジェクトの保存
const user = {
  name: '田中太郎',
  age: 25,
  hobbies: ['読書', '映画鑑賞', 'プログラミング']
};

// JSON形式で保存
localStorage.setItem('user', JSON.stringify(user));

// 取得時にオブジェクトに戻す
const savedUser = JSON.parse(localStorage.getItem('user'));
console.log(savedUser.name); // "田中太郎"
console.log(savedUser.hobbies); // ['読書', '映画鑑賞', 'プログラミング']
JSONの活用
  • JSON.stringify(): オブジェクト → JSON文字列
  • JSON.parse(): JSON文字列 → オブジェクト

この組み合わせでほとんどのデータ構造をlocalStorageに保存できます！
