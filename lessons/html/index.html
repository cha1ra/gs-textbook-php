HTMLの歴史

HTMLは1989年、ティム・バーナーズ＝リー博士が研究者たちが効率よく論文を検索・共有できるように発明したものです。文字に 「価値」 を与えて、タイトルやリンクなどの重要な情報を強調し、関連資料へすぐにアクセスできるようにするための仕組みがハイパーテキストとして誕生しました。

HTMLはプログラミング言語ではない！

実は、HTMLはプログラミング言語ではありません。HTMLはマークアップ言語と呼ばれる種類の言語です。

プログラミング言語との違い：

  • プログラミング言語（JavaScript、Python等）：コンピュータに「処理の手順」を指示する
    • 計算をする、条件によって処理を変える、繰り返し処理をする等
  • マークアップ言語（HTML）：文書に「意味や構造」を付ける
    • 「ここはタイトル」「ここは段落」「ここはリンク」等の情報を付与

ちなみに、CSSも スタイルシート言語 に分類されるため、プログラミング言語ではありません。
言葉の定義に気をつけましょう！

絶対パスと相対パス

リンク先や画像の読み込み元を指定する方法が2つあります。

絶対パス
定義: リソースの完全な位置を指定
例: https://www.example.com/page1.html
現実例: 東京都港区芝公園四丁目2番8号
相対パス
定義: 現在の位置を基準にして指定
例: ./images/logo.png
現実例: この道をまっすぐ進んで次の角を左
"現在の位置"とは？

Web開発における現在の位置とは、ファイルが存在するディレクトリ（フォルダ）のことです。

比較表：

項目絶対パス相対パス
定義リソースの完全な場所を指定するパス現在のページや文書を基準にしたリソースの場所
開始プロトコル（例: https://）から始まる./, ../、またはフォルダ/ファイル名
使用場面外部サイトや異なるドメインへのリンク同じプロジェクト内のリソース参照
例https://example.com./images/logo.png
画像 (<img>)
  • image の略
  • 画像をWebページに挿入する
<img src="cat.jpg" alt="かわいい猫の写真">

重要な属性：

  • src 属性：画像のパスを指定（source の略）
  • alt 属性：画像の説明を追加（alternative text の略、アクセシビリティのため）
補足 ブロックレベル要素とインライン要素
  • HTMLはブロックレベル、インラインのいずれかの性質を持つ
  • 大事なのは「改行の有無」！
ブロックレベル要素
役割: ページ全体の構造を作る
表示幅: 親要素の幅と同じ
改行: 改行が伴う
例: <div>, <h1>, <p>
インライン要素
役割: 文章の一部に意味を持たせる
表示幅: 内容の幅に応じて表示
改行: 改行は伴わない
例: <a>, <span>, <img>
厳密な分け方

厳密な分け方をするならコンテンツカテゴリという考え方がありますが、ややこしいので最初は上記２つの特性があるということがわかっていればOKです！

参考: https://developer.mozilla.org/ja/docs/Web/HTML/Content_categories

フォームの基本

フォーム (<form>)
  • ユーザーからの入力を収集するブロック
<form action="/submit" method="post">
  <!-- フォーム要素がここに入ります -->
</form>

action, method は サーバーサイド（PHP など）とやり取りするときに使います。今回の演習では省略して構いません。

input タグの3つの重要な属性
  • フォームの入力欄には、それぞれ3つの重要な情報を設定する
フォームタグの3つの属性
  • type: どんな種類の入力欄か（テキスト？パスワード？）
  • name: この入力欄の名前（データを識別するため）
  • value: 実際に入力された内容

例えば、ユーザー名の入力欄なら：

<input type="text" name="username" value="">
  • type="text" → 普通のテキスト入力
  • name="username" → この入力欄の名前は「username」
  • value="" → 最初は空、ユーザーが入力すると値が入る
テキスト入力（<input type="text">）

一行のテキスト入力用

<input type="text" name="username" placeholder="ユーザー名">
パスワード入力（<input type="password">）

パスワード入力用（文字が *** で表示される）

<input type="password" name="password" placeholder="パスワード">
選択肢の入力（<input type="radio"> と <input type="checkbox">）
ラジオボタン

複数の選択肢から1つを選ぶ

チェックボックス

複数の選択肢から複数選べる

ラジオボタンの例：

<input type="radio" name="gender" value="male"> 男性
<input type="radio" name="gender" value="female"> 女性

チェックボックスの例：

<input type="checkbox" name="hobby" value="reading"> 読書
<input type="checkbox" name="hobby" value="sports"> スポーツ
送信ボタン（<input type="submit">）

フォームのデータを送信する

<input type="submit" value="送信">

セマンティックHTMLのメリット
  • SEO向上: 検索エンジンが内容を理解しやすくなる
  • アクセシビリティ向上: スクリーンリーダーなどの支援技術が内容を正しく解釈できる
  • 保守性向上: コードの構造が明確になり、他の開発者が理解しやすくなる
