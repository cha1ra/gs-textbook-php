Cascading

カスケード = 滝

親から子へ、上から下へ、スタイルが「滝のように流れ落ちる」仕組み

例：親要素が「文字色：青」→ 子要素も自動的に「文字色：青」になる

Style Sheets

スタイル設計書

色・サイズ・配置などの見た目を指定する設計書

p { color: red; }
CSSの歴史

CSSは1994年、ハーコン・ウィウム・リー氏によって提案されました。当時のWebページはHTMLだけで作られており、見た目もすべてHTMLタグで指定していました。

昔のHTMLでの見た目指定例：

<!-- 文字色を変える -->
<font color="red">赤い文字</font>

<!-- 文字サイズを変える -->
<font size="5">大きな文字</font>

<!-- 背景色を変える -->
<body bgcolor="yellow">

このように、HTMLの中に見た目の情報が混在していたため、コードが複雑になり管理が大変でした。CSSの登場により、構造（HTML） と 見た目（CSS） を分離できるようになり、より効率的で美しいWebサイトが作れるようになったのです。

CSSの利点
HTMLとスタイルの分離
コードの保守性が高くなり、管理が容易
スタイル変更の容易さ
一箇所の変更で複数ページのデザイン変更が可能
効率的な開発
複数のHTMLページで同じスタイルを再利用可能
高速な表示
スタイルが適用される前のHTML文書の表示が速くなる

補足 @charset "utf-8" について
  • CSSファイルがUTF-8でエンコードされている場合に必要な記述
  • しかし、ほとんどの場合、HTMLのcharset属性やWebサーバーの設定でUTF-8を指定するため、記述が必要になることはほとんどない
  • そのため、CSSファイルもUTF-8でエンコードされることが多く、現在では省略されることが一般的
/* 必要に応じて記述（通常は省略可能） */
@charset "utf-8";

p {
  color: red;
}
CSSの基本ルール

CSSのポイントは 「どこの」 「何を」 「どうする」 です！

セレクタで プロパティを 値に設定する

セレクタ{
プロパティ: 値;
プロパティ: 値;
}
セレクタ:「どこの」要素にスタイルを適用するか
プロパティ:「何を」変更するか（色、サイズ、位置など）
値:「どうする」か（赤色にする、大きくするなど）

実際の例：

/* セレクタとプロパティの例 */
p {
  color: red;           /* 文字色を赤に */
  background-color: yellow;  /* 背景色を黄色に */
  font-size: 18px;      /* 文字サイズを18pxに */
}

重要な概念

ページの見た目を整えるには「ボックスモデル」の理解が必要不可欠です。今回はmarginとpaddingの違いに着目しながら学習しましょう！

margin の使い方

marginとは？

  • 要素の外側の余白を設定するプロパティ
  • 他の要素との間隔を空けるために使用
  • 「この要素の周りにスペースを作りたい」時に使う
marginの役割

marginは「要素と要素の間の距離」を決めます。例えば、段落と段落の間に空白を作ったり、ボタンを他の要素から離したりする時に使います。

書き方パターン
margin: 10px;
全方向に10px
margin: 10px 20px;
上下10px、左右20px
margin: 10px 20px 30px;
上10px、左右20px、下30px
margin: 10px 20px 30px 40px;
上→右→下→左（時計回り）
個別指定
margin-topmargin-rightmargin-bottommargin-left
よく使う値
margin: 0; - 余白なし
margin: auto; - 自動調整
margin: 0 auto; - 中央寄せ
margin: 20px; - 全方向20px
覚え方のコツ

時計回り（上→右→下→左）で覚えましょう！「上から右に回って下、最後に左」です。

主な用法：

  1. 要素間の間隔調整

    p {
  margin-bottom: 20px; /* 段落の下に20pxの間隔 */
}

  2. 中央寄せ

    .container {
  margin: 0 auto; /* 左右中央に配置 */
}

  3. レイアウト調整

    .header {
  margin-top: 50px; /* ヘッダーの上に50pxの余白 */
}
padding の使い方

paddingとは？

  • 要素の内側の余白を設定するプロパティ
  • コンテンツと境界線（border）の間の空間を作る
  • 「この要素の中身にゆとりを持たせたい」時に使う
paddingの役割

paddingは「要素の内側のスペース」を決めます。例えば、ボタンの文字と枠の間に余白を作ったり、カードの中身に余裕を持たせたりする時に使います。

marginとpaddingの文法は一緒です！

書き方パターン（marginと同じ）
padding: 10px;
全方向に10px
padding: 10px 20px;
上下10px、左右20px
padding: 10px 20px 30px;
上10px、左右20px、下30px
padding: 10px 20px 30px 40px;
上→右→下→左（時計回り）
個別指定
padding-toppadding-rightpadding-bottompadding-left

横並びレイアウトの主流

Flexboxは横並びのレイアウトを作成したいときに使われます。以前はfloat:leftが使われていたが今はFlexboxの方が主流です。

display:flexを適用すると横並びに！

box
縦に並んでいる
適用後（display: flex）
box
box
box
横に並んでいる

HTMLコード：

<div class="container">
  <div class="box">box</div>
  <div class="box">box</div>
  <div class="box">box</div>
</div>

CSSコード：

/* 横並びしたい要素をラップしているブロックに適用する */

  background-color: #e74c3c;
  color: white;
  padding: 12px;
  margin: 4px;
}

リセットCSS

ブラウザのデフォルトCSSの問題
ブラウザのデフォルトCSSの問題

各ブラウザはそれぞれ独自のCSSを持っています
そのため、CSSを指定しないとブラウザによって表示が異なるのです！

ブラウザのデフォルトCSS

  • Chrome: Chromium User Agent Stylesheet
  • Safari: WebKit User Agent Stylesheet

リセットCSS・ノーマライズCSS

  • The New CSS Reset: elad2412/the-new-css-reset
  • Normalize.css: necolas/normalize.css
問題点
• ブラウザごとに異なるスタイル
• 意図しない余白やスタイル
• デザインの一貫性が保てない
• 予期しない表示崩れ
リセットCSSの効果
• 全ブラウザで統一された表示
• クリーンな状態から開始
• 予測しやすいスタイル適用
• モダンな開発手法に対応
